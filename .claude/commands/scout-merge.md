여러 OSS 레포를 하나의 프로젝트로 통합합니다.

통합 대상: $ARGUMENTS

## 6단계 파이프라인

### 1단계: 레포 분석
각 레포의 기술 스택, 의존성, 구조를 파악합니다.
- `explain_repo`로 각 레포의 상세 정보를 수집합니다.
- 공통 기술과 차이점을 식별합니다.

### 2단계: 라이선스 호환성 검사
- `check_license`로 각 레포의 라이선스를 확인합니다.
- 라이선스 간 호환성을 검증합니다 (예: MIT + GPL = GPL 전파).
- 비호환 라이선스 발견 시 경고하고 사용자 판단을 요청합니다.

### 3단계: 의존성 분석
- 각 레포의 패키지 매니저와 의존성을 수집합니다.
- 버전 충돌을 감지합니다.
- 호환 가능한 버전을 결정합니다.

### 4단계: 통합 계획 수립
- 베이스 레포를 선정합니다 (가장 포괄적인 것).
- 다른 레포에서 가져올 기능/코드를 목록화합니다.
- 디렉토리 구조 통합 방안을 설계합니다.

### 5단계: 통합 실행
- `scaffold`로 베이스 레포를 생성합니다.
- 다른 레포의 코드를 수동으로 통합합니다.
- 설정 파일을 머지합니다.
- import 경로를 조정합니다.

### 6단계: 검증
- 의존성 설치 확인
- 빌드 성공 확인
- 기본 테스트 실행

## 규칙

- 각 단계 완료 시 사용자에게 진행 상황을 보고합니다.
- 3단계에서 심각한 충돌 발견 시 사용자 확인 후 진행합니다.
- 5단계 실행 전 통합 계획에 대한 사용자 승인을 받습니다.
- 모든 원본 LICENSE 파일을 보존합니다.
- 모든 응답은 한국어로 작성합니다.
