OSS Scout 전체 워크플로우를 오케스트레이션합니다.

사용자 요청: $ARGUMENTS

## 워크플로우 (4단계 파이프라인)

### 1단계: 검색 (Scout Researcher)
`search_boilerplate` 도구를 사용하여 사용자 요청에 맞는 보일러플레이트를 검색합니다.
- 쿼리에서 기술 스택과 기능 키워드를 추출합니다.
- 최소 5개 후보를 수집합니다.

### 2단계: 검증 (Scout Reviewer)
검색된 상위 3개 후보에 대해 `validate_repo`와 `check_license`를 실행합니다.
- 각 후보에 SAFE/CAUTION/RISK 판정을 부여합니다.
- RISK 판정을 받은 후보는 제외합니다.

### 3단계: 분석 (Scout Architect)
검증을 통과한 후보에 대해 `explain_repo`로 상세 분석합니다.
- 기술 스택, 아키텍처, 적합성을 평가합니다.
- 최종 추천 1-2개를 선정합니다.

### 4단계: 스캐폴딩 (Scout Builder)
사용자가 선택하면 `scaffold`로 프로젝트를 생성합니다.
- 이 단계는 사용자 확인 후에만 실행합니다.

## 출력 형식

각 단계 완료 시 진행 상황을 보고합니다:

```
[1/4] 검색 완료 - {n}개 후보 발견
[2/4] 검증 완료 - SAFE: {n}, CAUTION: {n}, RISK: {n}
[3/4] 분석 완료 - 최종 추천: {repo}
[4/4] 대기 중 - 스캐폴딩을 진행할까요? (대상 디렉토리를 지정해주세요)
```

모든 응답은 한국어로 작성합니다.
