자연어 요구사항에서 OSS 조합을 찾고 프로젝트를 구현하는 킬러 커맨드입니다.

요구사항: $ARGUMENTS

## 8단계 파이프라인

### 0단계: 기존 스펙 확인
먼저 `project-spec.md` 파일이 현재 디렉토리에 존재하는지 확인합니다.
- 파일이 존재하면, 해당 파일을 읽고 그 요구사항을 사용합니다 (1단계를 건너뜁니다).
- 파일이 존재하지 않으면, 1단계로 진행하여 $ARGUMENTS를 직접 분석합니다.
- $ARGUMENTS가 .md 파일 경로를 참조하면, 해당 파일을 스펙으로 읽습니다.

### 1단계: 요구사항 분해
자연어 입력에서 기능 요구사항을 추출합니다.
- 핵심 기능 (must-have)
- 부가 기능 (nice-to-have)
- 기술 스택 선호
- 비기능 요구사항

### 2단계: 기능별 OSS 검색
각 기능에 맞는 보일러플레이트를 `search_boilerplate`로 검색합니다.
- 풀스택 보일러플레이트 검색
- 기능별 특화 라이브러리 검색

### 3단계: 후보 검증
상위 후보에 대해 `validate_repo`와 `check_license`로 검증합니다.
- RISK 판정 후보는 자동 제외

### 4단계: 최적 조합 선정
검증된 후보 중 최적의 OSS 조합을 선정합니다.
- 기능 커버리지 최대화
- 기술 스택 호환성
- 라이선스 호환성

### 5단계: 사용자 확인
선정된 조합과 실행 계획을 사용자에게 제시합니다.
- **반드시 사용자 승인 후 다음 단계로 진행합니다.**

### 6단계: 스캐폴딩
승인된 베이스 레포를 `scaffold`로 생성합니다.

### 7단계: 통합 및 커스터마이징
추가 레포/라이브러리의 코드를 통합합니다.
- 의존성 추가
- 설정 파일 수정
- 기능 코드 이식

### 8단계: 검증 및 안내
최종 프로젝트의 빌드/실행을 검증하고 다음 단계를 안내합니다.
- 의존성 설치 확인
- 빌드 성공 확인
- 환경 변수 설정 안내
- 개발 서버 실행 방법 안내

## 규칙

- 5단계에서 반드시 사용자 승인을 받습니다.
- 배포 명령은 실행하지 않습니다.
- 시크릿은 하드코딩하지 않습니다.
- LICENSE 파일은 보존합니다.
- 모든 응답은 한국어로 작성합니다.
